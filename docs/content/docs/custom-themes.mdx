## Creating a Custom Theme

Custom themes are JSON files that define your color palette. Here's the complete structure:

<Tabs items={['Minimal', 'Full Example']}>
  <Tab value="Minimal">
    Start with just the essentials:

    ```json title="my-theme.json"
    {
      "id": "my-theme",
      "name": "my-theme",
      "displayName": "My Theme",
      "isDark": true,
      "colors": {
        "background": "#1a1b26",
        "foreground": "#c0caf5",
        "primary": "#7aa2f7",
        "primaryForeground": "#1a1b26"
      }
    }
    ```

    <Callout type="info">
      Missing color properties will inherit from the default theme.
    </Callout>

  </Tab>

## Theme Schema

### Metadata

| Property      | Type      | Required | Description                                |
| ------------- | --------- | :------: | ------------------------------------------ |
| `id`          | `string`  |   Yes    | Unique identifier (kebab-case recommended) |
| `name`        | `string`  |   Yes    | Internal name, typically matches `id`      |
| `displayName` | `string`  |   Yes    | Human-readable name shown in UI            |
| `description` | `string`  |    No    | Brief description of the theme             |
| `author`      | `string`  |    No    | Theme creator's name                       |
| `version`     | `string`  |    No    | Semantic version (e.g., `1.0.0`)           |
| `isDark`      | `boolean` |   Yes    | Set `true` for dark themes                 |
| `isBuiltIn`   | `boolean` |    No    | Always `false` for custom themes           |

### Color Properties

<Tabs items={['Core', 'UI Elements', 'Sidebar', 'Charts']}>
  <Tab value="Core">
    The fundamental colors that define your theme's personality.

    | Property | Purpose |
    | --- | --- |
    | `background` | Main application background |
    | `foreground` | Primary text color |
    | `primary` | Accent color for buttons, links, focus states |
    | `primaryForeground` | Text on primary-colored elements |
    | `secondary` | Secondary actions and less prominent UI |
    | `secondaryForeground` | Text on secondary elements |
    | `muted` | Subtle backgrounds, disabled states |
    | `mutedForeground` | Subtle text, placeholders |
    | `accent` | Highlights, hover states |
    | `accentForeground` | Text on accent elements |
    | `destructive` | Dangerous actions, errors, delete buttons |

  </Tab>
  <Tab value="UI Elements">
    Colors for specific interface components.

    | Property | Purpose |
    | --- | --- |
    | `card` | Card and panel backgrounds |
    | `cardForeground` | Text within cards |
    | `popover` | Dropdown and popover backgrounds |
    | `popoverForeground` | Text in popovers |
    | `border` | Borders and dividers |
    | `input` | Input field borders |
    | `ring` | Focus ring around interactive elements |

  </Tab>
  <Tab value="Sidebar">
    The sidebar has its own color system for visual separation.

    | Property | Purpose |
    | --- | --- |
    | `sidebar` | Sidebar background |
    | `sidebarForeground` | Sidebar text |
    | `sidebarPrimary` | Active/selected items |
    | `sidebarPrimaryForeground` | Text on active items |
    | `sidebarAccent` | Hover state background |
    | `sidebarAccentForeground` | Hover state text |
    | `sidebarBorder` | Sidebar borders |
    | `sidebarRing` | Focus ring in sidebar |

  </Tab>
  <Tab value="Charts">
    Five colors for data visualization consistency.

    | Property | Suggested Use |
    | --- | --- |
    | `chart1` | Primary data series |
    | `chart2` | Secondary data series |
    | `chart3` | Tertiary data series |
    | `chart4` | Fourth data series |
    | `chart5` | Fifth data series |

    <Callout type="info">
      Choose colors with good contrast between each other for readable charts.
    </Callout>

  </Tab>
</Tabs>

## Import & Export

<Steps>
<Step>

### Import a Theme

Open **Settings** and navigate to **Appearance**. Click **Import Theme** and paste your JSON or select a file.

</Step>
<Step>

### Apply Your Theme

Your imported theme appears in the theme list. Click to activate it instantly.

</Step>
<Step>

### Export for Sharing

Select any theme and click **Export Theme** to download the JSON file. Share it with others or back it up.

</Step>
</Steps>

## Pro Tips

<Accordions type="single">
  <Accordion title="Start from an existing theme">
    Export a built-in theme you like, then tweak the colors. This ensures you have all properties
    defined and a solid starting point.
  </Accordion>
  <Accordion title="Test accessibility">
    Use a contrast checker to ensure text remains readable. Aim for at least 4.5:1 contrast ratio
    for body text.
  </Accordion>
  <Accordion title="Test all UI states">
    Preview your theme with tables, charts, modals, and error states to catch any issues before
    sharing.
  </Accordion>
  <Accordion title="Set isDark correctly">
    The `isDark` flag affects scrollbars, system UI elements, and certain component behaviors. Dark
    background themes should set this to `true`.
  </Accordion>
</Accordions>
